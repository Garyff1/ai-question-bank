## 1. 本阶段完成内容

1. 补充试卷、错题、统计和富内容区域的深色主题与中英文适配。
2. 完成本地 OCR 资料闭环：拍照或相册多选、裁剪与旋转、本地识别、文本编辑、保存为资料并用于后续出题。

5. 建立统一 TTS/音频服务，提供播放、暂停、停止、重播、语速与错误降级。

7. 增加 `THIRD_PARTY_NOTICES.md`、README 第三方项目说明和 App 内开源许可入口。
8. 在模拟器冒烟中发现并补齐“管理资料 → 扫描资料”入口，确保 OCR 不只是存在代码中，而是用户可实际进入。

旧 JSON 缺少字段时使用默认值，不影响读取。

### 图表数据

新增统一可选结构：

```json
{
  "chartType": "line",
  "title": "温度变化",
  "xLabels": ["1月", "2月", "3月"],
  "series": [
    {"name": "温度", "values": [12, 15, 18]}
  ],
  "unit": "℃",
  "description": ""
}
```

解析器继续兼容旧版 `data: "甲:10,乙:20"` 格式。所有新增字段均可选，并提供空值与异常数据降级。

## 6. OCR 验收结果

已在 Android API 36 模拟器完成真实闭环：

1. 从“管理资料”进入“扫描资料”。
2. 打开 Android Photo Picker。
3. 选择本地截图。
4. 打开裁剪与旋转界面并确认。
5. ML Kit 成功识别截图中的英文内容。
6. 识别结果可编辑。
7. 保存后生成普通 `StudyMaterial`，并自动成为当前练习资料。
8. 全过程无 Fatal Exception。

安全降级：取消选择、裁剪失败、识别失败、空识别结果均显示提示，不创建空资料、不导致崩溃。

仍需体验验收：不同品牌真机的相机权限、中文手写笔记、倾斜/低光照片和多页大批量识别。

## 7. 图表验收结果

- 支持结构化折线图、柱状图、饼图。
- 支持旧版图表数据兼容。
- 无效数据返回占位状态，不抛异常。
- 答题/解析界面使用 `StructuredChartWidget` 本地绘制。
- PDF 使用同一结构化数据绘制真实图表，不输出 JSON/代码文本。
- `critical_regression_test` 已验证 PDF 图表绘制和“不出现原始代码”。

专项测试：`chart_data_test.dart` 3 项通过，原有媒体卡片回归测试通过。

## 8. TTS 验收结果

- 新增统一 `ListeningAudioService`。
- 支持播放、暂停、停止、重播和三档语速。
- 支持 locale 标准化与文本回退。
- TTS 引擎异常被捕获并显示提示，不阻塞答题。
- Android Manifest 已声明系统 TTS 服务查询。

兼容说明：普通听力题已使用 `flutter_tts`；旧试卷 MP3 导出暂时保留 `flutter_edge_tts` 兼容路径，后续需单独迁移与验证音频文件拼接质量。

## 9. 错题诊断验收结果

- 删除“错题知识点 Top5”条形图。
- 新增知识点诊断列表，显示错题数、正确率、连续答对次数、最近出错和掌握状态。
- 掌握规则集中在单一服务中：需要巩固、正在进步、基本掌握。
- 支持按掌握最弱、错题最多、最近出错排序。
- 提供开始巩固、查看错题、查看解析入口。

专项测试：`knowledge_diagnosis_test.dart` 通过。

## 13. 当前遗留问题

1. `main.dart` 仍然较大，Phase 2 新模块已拆分，但旧页面尚未完成全面模块化。
2. 深层旧页面仍有部分写死颜色和中英文混排；主流程可用，但需要继续做视觉走查。
3. App 英文模式下，Android 裁剪器界面可能跟随系统/第三方组件语言显示中文。
4. `flutter_edge_tts` 仍用于旧试卷 MP3 导出兼容路径。
5. 中文 OCR 模型使 APK 增长约 31.55 MiB。
6. `syncfusion_flutter_pdf` 存在商业许可边界，正式收费前必须解决。
7. 真机相机、中文手写、弱光和多页大资料仍需专项体验测试。
8. 当前 29 条 analyze info 为旧代码风格提示，无 error/warning；本阶段没有以大面积重写方式清理。
