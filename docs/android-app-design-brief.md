# AI题库 Android App 整体设计文档

> 面向设计师 / UIUX 设计协作使用  
> 当前版本：Android v1.0.4 早期测试版  
> 项目定位：把课件、笔记、PDF、Word、TXT 等学习资料转成可练习、可复盘的 AI 出题工具。

---

## 1. 项目一句话

AI题库是一款面向学生、教师、备考人群和自学用户的本地化 AI 出题 App。用户导入学习资料后，配置自己的大模型 API Key，App 会根据资料生成选择题、判断题、填空题、主观题和混合练习，并自动记录练习历史与错题。

Android 版是“单体 App”：

- 不需要注册 / 登录云端账号。
- 不需要用户手动启动电脑后端。
- 资料、API Key、练习记录、错题本都保存在当前手机本地。
- App 直接调用用户自己配置的大模型接口。

## 2. 设计目标

### 2.1 产品目标

让用户从“手里有资料”到“开始刷题复习”的路径尽可能短：

```text
导入资料 → 配置 API Key → 选择题型 → 生成题目 → 答题 → 错题复习
```

### 2.2 体验目标

1. 首次使用不迷路  
   用户第一次打开 App 时，需要知道底部每个入口分别做什么。

2. API 配置不吓人  
   大部分用户不知道 API Key 是什么、去哪里申请、Base URL 和模型名怎么填。设计上要把“配置模型”从工程感很强的表单，变成清晰、可引导的步骤。

3. 操作反馈更明确  
   切换页面、保存配置、生成题目、导入资料、删除资料、清空错题等行为都要有明确反馈，减少用户误解和焦虑。

4. 保持轻量学习工具气质  
   不做复杂教育平台，不做重运营社区。整体应该像一个清爽、可信、专注的“学习助手”。

## 4. 目标用户

### 4.1 学生 / 备考用户

典型需求：

- 把课堂 PDF、复习资料、笔记转成练习题。
- 用碎片时间刷题。
- 自动收集错题，考前复习。

痛点：

- 自己整理题目很麻烦。
- 资料很多，不知道怎么复习。
- 不熟悉 API Key 和模型配置。

### 4.2 教师 / 培训人员

典型需求：

- 快速从课件生成课堂练习。
- 根据资料生成不同题型。
- 检查学生掌握情况。

痛点：

- 手动出题耗时。
- 题型需要多样化。
- 希望流程简单、稳定。

### 4.3 自学用户 / 职业考试用户

典型需求：

- 导入教材、讲义、考试资料。
- 根据资料生成专项练习。
- 反复训练错题。

痛点：

- 资料长，自己抓重点效率低。
- 希望能按资料归档错题。
- 希望离线保存自己的学习记录。

## 5. 信息架构

Android App 当前底部导航共 5 个一级入口：

```text
资料 / 出题 / 错题 / 统计 / 配置
```

### 5.1 资料

作用：

- 导入 PDF、DOCX、TXT、Markdown、CSV、JSON 等资料。
- 查看已导入资料列表。
- 选择当前用于出题的资料。
- 删除资料。

设计重点：

- “导入资料”应是最强主操作。
- 资料卡片要清楚显示文件名、摘要、导入时间、文件类型。
- 删除按钮需要二次确认，避免误触。
- 大文件导入需要进度反馈，不能让用户误以为 App 卡死。

### 5.2 出题

作用：

- 选择资料。
- 选择题型，可多选。
- 选择目标群体。
- 调整题目数量。
- 点击生成题目。

题型：

- 单选题
- 多选题
- 判断题
- 填空题
- 主观题

设计重点：

- 题型选择建议采用大卡片 / Chip 组合，明确“可多选”。
- 生成按钮应有足够视觉权重。
- 未选择资料、未配置 API、正在生成、生成失败等状态需要清晰提示。

### 5.3 错题

作用：

- 展示答错的题。
- 支持清空错题。
- 未来需要支持错题训练。
- 未来需要按资料分组收纳错题。

设计重点：

- 清空错题必须二次确认。
- 错题应显示来源资料，避免多个资料混在一起后用户不知道题目来自哪里。
- 建议增加“开始错题训练”主按钮。
- 未来可做“全部错题 / 按资料 / 按题型 / 最近错误”筛选。

### 5.4 统计

作用：

- 展示累计做题数。
- 展示正确率。
- 展示错题数量。
- 展示历史练习记录。

设计重点：

- 统计页应偏“轻仪表盘”，不要过度复杂。
- 数字卡片可增加趋势感、图标感。
- 历史记录要能帮助用户回忆“哪份资料练过、得分如何”。

### 5.5 配置

作用：

- 选择大模型服务商。
- 填写 API Key。
- 填写 Base URL。
- 填写模型名称。
- 测试连接。
- 保存配置。

当前预设服务商：

| 服务商 | Base URL | 推荐模型 | 备注 |
| --- | --- | --- | --- |
| DeepSeek | `https://api.deepseek.com` | `deepseek-v4-flash` | OpenAI-compatible |
| Qwen / 阿里百炼 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` | OpenAI-compatible |
| 智谱 / Z.ai | `https://api.z.ai/api/paas/v4` | `glm-4.5-flash` | OpenAI-compatible |
| 小米 MiMo | `https://api.xiaomimimo.com/v1` | `mimo-v2.5-pro` | 使用 `api-key` Header |
| Kimi / Moonshot | `https://api.moonshot.ai/v1` | `kimi-k2.6` | OpenAI-compatible |
| 自定义 | 用户填写 | 用户填写 | 兼容 Chat Completions 的模型服务 |

设计重点：

- 这是最容易劝退用户的页面，需要设计成“引导式配置”。
- 建议分为：
  1. 选择服务商
  2. 获取 API Key 指引
  3. 填写并测试
  4. 配置成功状态
- 配置成功后建议出现更强反馈，例如“电池充电动画 / 已接入模型引擎”。
- 若用户想换服务商，下方提供“更换其他接口”按钮。

## 6. 首次使用引导页

### 6.1 当前已加入的引导内容

首次打开 App 时，会出现 4 步引导：

1. 先导入资料
2. 再生成题目
3. 错题会自动收集
4. 最后配置 API Key

### 6.2 设计优化建议

引导页可以更像“产品教学”，不是普通说明弹窗：

- 使用品牌插画或小动画。
- 每一步只讲一个动作。
- 让用户知道“下一步该点哪里”。
- 最后一页可以放两个按钮：
  - 立即配置 API
  - 先导入示例资料

### 6.3 推荐文案

```text
欢迎使用 AI题库
把你的资料变成练习题。

1. 导入资料
上传 PDF、Word、TXT 或直接粘贴笔记。

2. 选择题型
单选、多选、判断、填空、主观题都可以混合生成。

3. 开始练习
AI 根据资料生成题目，答完自动记录练习结果。

4. 复习错题
答错的题会自动进入错题本，方便考前巩固。
```

## 7. 核心用户流程

### 7.1 新用户首次完整流程

```text
打开 App
→ 看首次引导
→ 进入配置页
→ 选择服务商
→ 查看官网 API Key 指引
→ 填入 API Key
→ 测试连接成功
→ 导入资料
→ 选择题型和数量
→ 生成题目
→ 答题
→ 查看解析
→ 错题进入错题本
```

### 7.2 老用户日常流程

```text
打开 App
→ 选择已有资料
→ 生成题目
→ 答题
→ 看错题 / 看统计
```

### 7.3 配置异常流程

常见错误：

- 没填 API Key。
- Key 复制不完整。
- Base URL 填错。
- 模型名填错。
- 服务商账号余额不足。
- 网络无法访问模型服务商。

设计建议：

- 错误提示不要只写“请求失败”。
- 应给出下一步建议：
  - “请检查 API Key 是否复制完整”
  - “请确认模型名是否与服务商控制台一致”
  - “请确认账号余额或免费额度是否可用”
  - “可前往官网查看配置教程”

## 8. 页面设计建议

### 8.1 全局布局

建议采用：

- 顶部：页面标题 + 简短说明
- 中部：核心内容卡片
- 底部：主操作按钮或底部导航

整体留白要充足，避免表单和卡片挤在一起。

### 8.2 卡片系统

建议卡片区分 4 类：

| 类型 | 用途 | 视觉建议 |
| --- | --- | --- |
| 功能卡片 | 资料、题型、服务商选择 | 白底、圆角、轻阴影 |
| 状态卡片 | API 已配置、导入成功、生成中 | 可使用浅色背景和状态图标 |
| 危险操作卡片 | 删除资料、清空错题 | 红色轻提示，二次确认 |
| 空状态卡片 | 暂无资料、暂无错题、暂无记录 | 插画 + 引导按钮 |

### 8.3 按钮层级

| 层级 | 用途 | 示例 |
| --- | --- | --- |
| 主按钮 | 当前页面最关键动作 | 生成题目、保存配置、导入资料 |
| 次按钮 | 辅助动作 | 测试连接、更换接口 |
| 文本按钮 | 低权重动作 | 跳过、查看教程 |
| 危险按钮 | 删除 / 清空 | 清空错题、删除资料 |

### 8.4 底部导航

当前底部导航：

- 资料
- 出题
- 错题
- 统计
- 配置

已加入轻量震动反馈。设计上建议：

- 当前选中项更明显。
- 图标风格统一，不要有的偏线性、有的偏实体。
- “出题”可以稍微强化，因为它是核心转化入口。

## 11. API 配置页设计重点

这是最需要设计优化的页面。

### 11.1 用户困惑点

用户可能不知道：

- API Key 是什么。
- 去哪里申请。
- 为什么要自己配置。
- Base URL 是什么。
- 模型名称为什么不能随便填。
- 测试失败后该怎么处理。

### 11.2 推荐页面结构

```text
顶部状态区
  - 当前是否已配置
  - 当前服务商
  - 当前模型
  - 成功状态动画

服务商选择区
  - DeepSeek / Qwen / 智谱 / 小米 MiMo / Kimi / 自定义

获取 Key 指引区
  - “不知道 API Key 在哪里获取？”
  - “打开官网教程”

配置表单区
  - API Key
  - Base URL
  - 模型名称

操作区
  - 测试连接
  - 保存配置
  - 更换其他接口
```

### 11.3 成功状态建议

可用一个轻量拟物隐喻：

```text
模型电池已接入
当前服务商：DeepSeek
当前模型：deepseek-v4-flash
```

动画建议：

- 电池充电流动效果。
- 蓝色能量条。
- 成功后短暂震动 / 勾选动画。

## 12. 导入资料体验优化

当前问题：

资料越大，导入时卡顿越久，严重时可能自动退出。

设计侧建议：

1. 导入前提示：

```text
正在解析资料，较大的 PDF 可能需要几十秒。
```

2. 导入中状态：

```text
读取文件 → 提取文本 → 清理内容 → 保存资料
```

3. 大文件提示：

```text
文件较大，建议保持 App 前台运行。
```

4. 失败兜底：

```text
这份资料解析失败。你可以尝试复制文本内容导入，或换成 TXT / DOCX 格式。
```

## 18. 给设计师的核心提醒

这不是一个普通文件管理 App，也不是传统题库 App。它的核心价值是：

```text
用户自己的资料 + 用户自己的 AI Key → 自动生成专属练习
```

所以设计要重点解决两个问题：

1. 让用户相信“我的资料可以安全地存在本地，并被转化成练习”。
2. 让用户不害怕“API Key 配置”这件事。
