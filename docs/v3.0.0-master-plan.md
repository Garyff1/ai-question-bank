# AI题库 v3.0.0 总体规划

## 1. 产品目标

AI题库 v3.0.0 将现有“资料生成题目工具”升级为个人 AI 学习训练与闯关平台。核心链路保持为：资料导入 → AI 出题 → 练习 → 错题沉淀 → 巩固与闯关 → 学习诊断 → XP 与打卡激励。

公开稳定版继续保持 Android v2.9.9；v3.0.0 在独立开发分支按阶段构建和验收，未经确认不发布 Release、不切换官网安装包。

## 2. 当前基线审计

- Android 版本：`2.9.9+50`。
- Android 主文件 `lib/main.dart` 约 2 万行，包含模型、服务、页面、通用组件、PDF、富媒体和闯关逻辑。
- 本地数据使用 SharedPreferences，核心兼容键包括 `materials_v1`、`records_v1`、`wrongs_v1`、`papers_v1`、`xp_profile_v1`、`practice_log_v1` 和 `rpg_progress_v1`。
- API Key 当前包含在 `api_config_v1` 中，仍属于普通本地存储；安全存储迁移安排在第三阶段。
- 闯关已有 3 章 × 5 关、星级、XP、徽章和多种小游戏，但缺少统一的关卡规则、连击统计、知识护盾语义和完整结果诊断。
- 桌面端由 Tauri 壳和内置 FastAPI sidecar 组成；v3 第一阶段优先统一产品规范和 Android 体验，桌面端在后续阶段同步。

## 3. 三阶段范围

### 第一阶段：基础架构与闯关体验

- 建立 `app/`、`core/`、`features/` 分层，逐步拆分超大入口文件。
- 建立统一颜色、排版、间距、圆角、组件和动效规范。
- 支持浅色、深色、跟随系统。
- 支持简体中文、English、跟随系统。
- 首页升级为今日学习工作台。
- 闯关增加关卡类型、知识护盾、连击、Boss反馈和完整结果页。
- 保持 v2.9.9 数据与主流程兼容。

### 第二阶段：资料与富媒体能力

- OCR 扫描资料与可编辑识别结果。
- 结构化图表题及本地渲染。
- TTS/音频题稳定性重构。
- 错题知识点诊断列表。
- 第三方开源项目、许可证和合规说明。

### 第三阶段：官方 AI 与支付

- 自带 API Key 免费模式与官方 AI 服务双模式。
- API Key 安全存储与服务端密钥隔离。
- 影子计费、订单、微信支付、支付宝、退款和消费记录。
- 隐私政策、用户协议、付费协议及数据删除流程。

## 4. 功能依赖关系

1. 主题、语言和设置控制器是所有新页面的基础。
2. 统一组件和路由基础完成后，页面才能安全拆分。
3. OCR、图表和音频依赖第二阶段的富媒体模型整理。
4. 官方 AI 与支付依赖稳定的生成服务抽象、账户后端和合规文档。
5. 正式发布依赖三阶段总验收及 v2.9.9 数据迁移验证。

## 5. 数据迁移与风险

- 第一阶段不修改已有核心存储键；新增设置使用 `settings.*_v3` 命名。
- 新模型字段必须可选并提供默认值，旧 JSON 继续可读。
- API Key 暂不迁移，避免第一阶段引入认证数据风险；第三阶段再做安全迁移与回滚。
- 超大 `main.dart` 采用逐模块迁移，每次迁移后执行 analyze/test，禁止一次性重写。
- 主题迁移时保留闯关专用深色舞台，避免机械替换白色造成可读性问题。

## 6. 阶段验收标准

### 第一阶段

- 新应用壳、主题、国际化和设置模块可用。
- 三种主题、三种语言选择可持久化。
- 首页工作台和五类闯关规则可见。
- 闯关具备护盾、连击、Boss能量和诊断结果。
- 旧资料、错题、XP、练习和闯关数据可读取。
- `flutter analyze`、`flutter test`、Release APK 构建通过。

### 第二阶段

- OCR、图表、音频、诊断和开源许可均通过专项验收。

### 第三阶段

- 双 API 模式、影子计费、支付、退款、幂等和协议通过端到端验收。

## 7. v3.0.0 暂不包含

- iOS 正式发布。
- 复杂会员、钱包、充值赠送、连续包月和提现。
- 大型 3D 角色、视频关卡和高体积离线模型。
- 自动翻译用户资料或既有题目。
- 未经成本分析的无限云端 AI 调用。

## 8. 发布与回滚

- 开发版本使用 `3.0.0+51` 起递增构建号，产物仅供本地/分支验收。
- 正式确认前官网和 GitHub 最新公开下载仍为 v2.9.9。
- 每阶段单独提交并保留可构建节点。
- 若新设置或 UI 出现问题，可回退到上一阶段提交；核心 v2 数据不做破坏性迁移。
- 正式发布前提供 APK、SHA-256、变更日志、测试报告、迁移报告和已知问题清单。
