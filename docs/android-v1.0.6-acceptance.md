# AI题库 Android v1.0.6 验收说明（候选版）

> 状态：候选版 / 内测验收中  
> 结论：暂不发布 GitHub Release，暂不切换官网 `/download/android.apk`。  
> APK：`artifacts/ai-question-bank-android-v1.0.6.apk`  
> SHA-256：`E78F3703FE258A5F9047E2A4B749D249E1F603D728D19E137BCF76BDC2BD6976`

## 1. 本版定位

Android v1.0.6 主要完成的是界面结构优化候选版，重点参考设计稿优化了：

- 资料页：导入资料、粘贴资料、示例资料、资料卡片列表；
- 出题页：资料选择、题型多选、目标群体、题目数量；
- 错题本：错题总览、错题抽卡入口、按资料分组收纳；
- 学习统计：经验卡片、近 7 天练习趋势、统计卡片、练习历史；
- 安全交互：删除资料、清空错题增加二次确认；
- 触感反馈：底部导航、题型切换、数量调整等操作增加震动反馈。

本版没有对错题抽卡动画做专项升级，因此不建议直接作为正式公开版本发布。

## 2. 错题抽卡动画现状

当前 v1.0.6 保留了 v1.0.5 的基础抽卡体验，并没有做独立的动画系统重构。

目前已经具备：

- 全屏抽卡弹窗；
- 深色雾化背景；
- 5 张卡片从展开状态向中间聚拢；
- 抽取完成后的中心卡片高亮；
- 抽取成功文案；
- 成功时震动反馈；
- “跳过动画”按钮。

但以下能力还没有专项完成：

- 更明确的入场动画；
- 明显的洗牌动画；
- 独立的中央抽取动作；
- 真实翻卡动画；
- 更强的发光、粒子或奖励反馈；
- 动画节奏分段，例如“准备 → 洗牌 → 抽取 → 翻卡 → 成功”。

因此，如果验收重点是“抽卡动画更生动”，v1.0.6 还不能算完成。

## 3. 逐项验收结论

| 验收项 | 当前状态 | 说明 |
| --- | --- | --- |
| 入场动画 | 未专项优化 | 当前是弹窗直接进入，没有独立开场动效。 |
| 洗牌动画 | 未专项优化 | 只有卡片聚拢与轻微摆动，不是真正洗牌。 |
| 中央抽取 | 部分具备 | 动画结束后中间卡片高亮，但缺少“从牌堆抽出”的明确动作。 |
| 翻卡发光 | 部分具备 | 中间卡有高亮和阴影，但没有真实翻转。 |
| 抽取成功反馈 | 已具备 | 有“抽取成功”文案、开始挑战按钮和震动反馈。 |
| 跳过动画 | 已具备 | 点击“跳过动画”会直接进入挑战。 |
| 低端机性能风险 | 低 | 当前只是 5 张卡片的 Transform/AnimatedContainer，没有复杂粒子或视频。 |
| XP / 打卡 / 三倍经验 | 未改变 | v1.0.6 没有修改 XP 结算、打卡和三倍经验核心逻辑。 |

## 4. 必测清单

安装 v1.0.6 后，请优先测试：

1. 资料页是否整齐，导入入口是否清楚；
2. 导入文件是否仍可正常解析；
3. 粘贴资料是否仍可生成题目；
4. 示例资料是否仍可生成题目；
5. 出题页题型多选是否仍正常；
6. 题目数量加减是否正常；
7. 错题本是否按资料分组展示；
8. 错题抽卡入口是否仍可进入；
9. 抽卡动画体验是否达到预期；
10. 跳过动画是否能直接进入挑战；
11. 5 道错题全对是否仍能开启三倍经验；
12. XP 结算弹窗是否正常；
13. 删除资料是否出现二次确认；
14. 清空错题是否出现二次确认；
15. 底部导航切换是否有震动反馈；
16. 页面切换是否存在明显卡顿、闪屏或布局溢出。

## 5. 发布建议

当前建议：

```text
v1.0.5：XP + 错题抽卡基础版，已发布
v1.0.6：界面结构优化候选版，暂不发布
v1.0.7：错题抽卡动画专项优化版，完成后再考虑发布
```

如果 v1.0.6 真机体验稳定，但抽卡动画仍不满意，下一步应进入 v1.0.7：

- 增加完整抽卡流程：入场 → 洗牌 → 中央抽取 → 翻卡 → 发光 → 成功；
- 增加跳过动画但保持可控；
- 控制动画时长在 2.5—4 秒；
- 避免大量粒子、模糊或高频重绘，照顾低端安卓机；
- 不改动 XP、错题、三倍经验的数据逻辑。
