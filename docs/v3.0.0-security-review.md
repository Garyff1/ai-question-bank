# AI题库 v3.0.0 Phase 3 安全审查

## 1. 已完成的安全措施

### 客户端

- 用户 API Key 使用 Android Keystore 对应的安全存储。
- 旧 Key 只有在安全写入并回读一致后才删除明文。
- 迁移失败自动回滚，避免数据丢失。
- 官方登录 Token 与个人 API Key 分离；退出官方账号不删除个人 Key。
- 日志与异常信息不输出完整 Key 或 Authorization 请求头。
- Release 未启用本地明文 HTTP；明文访问只存在于 Debug Manifest。

### 服务端

- 官方订单、使用量、数据删除等接口要求认证并校验用户归属。
- 价格由服务端计算并在创建订单时复核。
- 订单使用 `clientRequestId` 和幂等键阻止重复创建、重复支付与重复退款。
- 支付回调抽象包含订单号、金额、状态和重复通知校验。
- 影子计费日志经过脱敏，不保存完整资料和 Prompt。
- 作者 API Key 和支付配置只通过环境变量占位，不进入仓库、APK或测试。
- `REAL_CHARGE_ENABLED` 默认关闭；真实支付适配器在未完成实现时强制不可用。
- Fake AI 和 Mock 支付用于自动化测试，不消耗真实额度。

## 2. 已测试场景

- 旧 Key 安全迁移、回读确认、失败回滚和删除。
- 官方 Token 401 清理与个人 Key 隔离。
- 报价过期、客户端篡改价格、重复创建和幂等键。
- 重复支付回调、金额不一致和非法状态转换。
- AI 超时、无效 JSON、服务商失败、自动重试和退款。
- 重复退款、越权订单查询、功能开关关闭。
- 日志脱敏、官方数据删除和账户注销。

## 3. 未解决风险

1. 当前开发数据库为 SQLite，正式环境需要 PostgreSQL、Alembic、最小权限账户、备份与恢复。
2. 开发任务使用本地后台任务，正式环境需要可靠队列、任务租约、超时和死信处理。
3. 尚未实现多实例共享的限流与防重放存储；正式环境需 Redis 或网关级限流。
4. 现有认证模块仍需生产密码策略、登录失败限流、Token 轮换和撤销机制审计。
5. 微信与支付宝的正式签名验签、证书轮换和回调源验证尚未实现。
6. 正式服务器、域名、HTTPS、WAF、数据库加密和密钥管理服务未配置。
7. 后端旧桌面 API 配置路径仍需单独审计，不能假设 Android Keystore 覆盖桌面端。
8. 法律与隐私草案尚未经过审核。

## 4. 发布建议

- 不在客户端开启真实支付开关。
- 生产密钥使用云密钥管理或受限环境变量，不写入 `.env` 文件镜像。
- 建立集中审计日志、告警和异常订单人工处理流程。
- 对登录、报价、订单、支付回调、退款和数据删除实施共享限流。
- 进行一次依赖漏洞扫描、渗透测试、密钥泄露扫描与数据删除演练。
- 完成所有商业发布阻断项前，不允许真实扣费。
