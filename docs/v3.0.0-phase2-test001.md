## 2. 本轮修复内容

1. OCR 取消裁剪后不再继续识别原图，失败页和取消页不会生成空资料。
2. 多页 OCR 只统计成功页，并按文件名自然顺序排列，修复 `PAGE-5` 到 `PAGE-1` 的倒序问题。
3. 图表数据增加有限数校验；饼图拒绝负数和总和为零的数据；PDF 坐标轴支持负数和零基线。
4. TTS 增加请求序号，快速连续点击时只保留最新播放请求；组件销毁后不再回调旧状态。
5. 错题知识诊断增加“最近练习”排序，形成四种排序方式。
6. “开始巩固”优先按完整题干检索；旧记录只有知识点标签时回退到资料名，实际复验由 `0/1` 恢复为 `1 条结果`。
7. 补齐试卷、错题、统计、API 配置、XP、反馈和折叠区的关键英文文案及深色主题颜色。
8. 试卷初始流程在英文界面显示英文科目、学段和考试类型，但继续保存原有中文内部值，避免破坏旧数据。
9. 关于页和开源许可页显示 `v3.0.0 Phase 2 Test001`。
10. 新增 OCR、图表、TTS、诊断排序和旧数据巩固跳转的专项自动化测试。

## 3. OCR 实际测试样本与结果

本轮使用 Android API 36 模拟器和相册选择器，准备了受控测试图片。测试图片位于本地构建目录，未提交到仓库。

| 样本 | 实际结果 | 识别耗时 | 主要误差 |
| --- | --- | ---: | --- |
| 清晰中文印刷资料 | 成功识别并进入编辑页 | 3599 ms | “所含”识别为“所合” |
| 中英文混排资料 | 中英文均可识别 | 未可靠记录 | “重要分支”截断为“重要” |
| 模拟手写中文 | 成功识别主要内容 | 3327 ms | “二次函数”识别为“二次函教”，`x²` 识别为 `x2` |
| 轻微倾斜（约 7°） | 成功识别全部行 | 3323 ms | “叶绿体”识别为“叶绿休”等少量错误 |
| 弱光与阴影模拟图 | 成功识别全部行 | 4345 ms | 可编辑修正 |
| 5 页连续资料 | 成功识别、编辑和保存 | 21482 ms | 修复后顺序为 `PAGE-1` 至 `PAGE-5` |

完整闭环已验证：

`相册选择 → 裁剪/旋转 → OCR → 文本编辑 → 保存资料 → 成为当前出题资料`

保存后的资料为“Scanned material 7/16”，共 281 个字符，证明资料保存和出题页资料交接正常。为避免消耗用户已配置 API 的额度，本轮没有实际调用第三方模型生成题目；“选择该资料并进入生成流程”已验证。

边界处理：

- 取消裁剪不会继续识别原图。
- 识别失败页不会计入保存页数。
- 全部页面失败时不创建空资料。
- 多页相册按文件名自然顺序处理。
- OCR 错误可在保存前编辑，不阻塞主流程。

注意：“手写中文”样本为楷体、旋转和噪声组合的模拟样本，不等同于真实手写笔记。真实手写、真实相机、切后台恢复和不同厂商 Photo Picker 仍列入真机待测。

## 4. 图表测试结果

已验证：

- 折线图、柱状图、饼图在中文/英文、浅色/深色四种组合中可渲染。
- 窄屏和长标签不触发布局异常。
- 旧版 `甲:10,乙:20` 形式仍可解析。
- 小数、负数和长标签可以进入统一结构；负数柱状图和折线图使用零基线。
- 饼图遇到负数、全零、非有限数或长度不一致时显示本地化占位，不崩溃。
- 答题页和解析页共用 `StructuredChartData`。
- PDF 回归测试确认图表使用同一结构化数据绘制，不输出原始 `rich_content` 代码。

自动化覆盖：`chart_data_test.dart`、`structured_chart_widget_test.dart`、`critical_regression_test.dart`。

## 5. TTS 测试结果

统一音频服务自动化验证通过：

- 中文、英文和中英混合 locale 归一化。
- 播放、暂停、停止、重播。
- 三档语速及异常语速钳制。
- 快速重复点击时只执行最新请求。
- TTS 不可用时返回错误状态，不抛出到答题流程。
- 页面销毁后取消待处理语音，不再通知已销毁组件。

模拟器以 `-no-audio` 启动，因此本轮不能声称真实扬声器输出已经通过。系统无语音、不同手机语音包、音量焦点、蓝牙耳机和真实中英文混读需真机验证。

旧试卷 MP3 导出仍保留 `flutter_edge_tts` 兼容路径。该路径需要网络和真实音频环境，本轮未实际导出，正式发布前必须单独回归。

## 6. 错题知识诊断结果

- 同一知识点聚合、错题数、正确率、最近出错、最近练习和连续答对计算通过单元测试。
- 掌握状态统一为“需要巩固 / 正在进步 / 基本掌握”。
- 四种排序均通过：掌握最弱、错题最多、最近出错、最近练习。
- “查看错题”和“查看解析”使用诊断项保留的数据。
- 无错误知识点时不生成诊断项，并显示空状态。
- 旧知识点条形图已移除；近 7 天学习趋势图仍保留。
- 模拟器实际点击“开始巩固”后，旧数据从 `0 of 1 mistakes found` 修复为 `1 results`，并显示正确资料和对应错题。

## 12. Syncfusion 正式发布处理建议

当前 PDF 能力依赖 Syncfusion。其商业授权不能简单继承仓库的 MIT 许可证。正式收费前必须完成以下一种方案：

1. 申请 Syncfusion Community License，确认项目和主体符合资格，并保存批准记录、条款版本和日期。
2. 购买商业许可证，并保存订单、授权范围和版本记录。
3. 在正式收费前替换 PDF 引擎，完成公式、分页、图表、中文字体和旧试卷回归。

本轮不擅自替换 PDF 方案。第三阶段可以进行架构设计，但正式收费发布必须先关闭该许可风险。
