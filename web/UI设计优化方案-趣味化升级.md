# AI题库 - 趣味化交互优化方案（设计补充文档）

> 基于原《UI设计需求文档.md》的视觉与动效升级，重点解决"不好看 / 互动性弱 / 趣味性不足"的问题。建议与原文档一起发给 UI 设计 AI 工具，作为补充规范使用。

---

## 一、问题诊断 & 核心理念

原方案的色彩、卡片、按钮规范都很扎实，问题在于**所有交互都是"静态切换"**——选中变色、提交变色、跳转翻页，没有一个能让人记住的瞬间，所以显得工具感强、趣味性弱。

**签名设计元素：判卷盖章**

把答题反馈设计成真实的"老师批改作业"动作：答对/答错的瞬间，一枚带毛边质感的红色/绿色"印章"从题卡上方"咚"地砸下来，伴随细小墨点飞溅；答错时题卡轻微抖动一下（不是惩罚感的剧烈晃动，是带点幽默的"哎呀"感）。这个动作复刻了所有中国学生最熟悉的"老师阅卷盖章"场景，比通用的勾叉动画更有记忆点，也呼应"上传纸质资料"这个产品核心载体。

配合这个隐喻，连带升级了几个关联组件：
- 答题进度条 → 一排"试卷标签"，每答完一题就盖上对应颜色的小标签
- 统计页大数字 → "成绩单"式环形进度 + 数字滚动递增
- 资料列表删除 → 左滑露出"撕纸"式删除按钮

## 二、新增 / 调整的动效与组件

| 场景 | 原方案 | 优化后 |
|------|--------|--------|
| 答题反馈 | 纯色背景条 + 文字 | 印章动画砸下 + 墨点飞溅；答错时题卡轻微抖动 |
| 答题进度条 | 单色细线 | 分段"试卷标签"，逐题点亮对/错颜色 |
| 连续答对 | 无 | 🔥 连胜计数器，随连胜值放大发光，答错时"啪"一下熄灭 |
| 题型选择卡片 | 静态边框变化 | 选中时做一次轻量翻转(flip)，带勾选图标弹出 |
| 数量计数器 +/- | 数字直接跳变 | 果冻式弹跳缩放；到边界时左右抖动提示，而非直接置灰 |
| 资料列表删除 | 系统弹窗直接删 | 左滑露出"撕纸"式删除按钮，确认后卷收动画消失 |
| 上传资料 | 顶部 Banner 文字 | Banner + 新卡片从列表顶部弹入(pop)，带成功提示色变化 |
| 答题完成页 | 静态百分比数字 | 彩色纸屑飘落 + 环形进度绘制动画 + 分数滚动递增 |
| 登录页 Logo | 静态文字 | 四个字逐字"盖章"式落下，带轻微回弹 |
| 空状态 | 纯文案 | 图标轻微浮动（呼吸感），文案保持原有引导语气 |
| 底部导航 | 选中态变色 | 选中背景做胶囊滑块过渡 + 点击轻微弹跳 |

## 三、各页面优化要点

| 页面 | 优化重点 |
|------|----------|
| 登录页 | Logo 盖章式入场 + 背景两个柔和墨渍光斑缓慢漂移，避免大片纯色背景的单调感 |
| 资料列表 | 文件类型图标改为圆形色块（PDF/Word/TXT 配色保留），列表项支持真实拖拽滑动删除 |
| 生成题目 | 题型卡片翻转反馈 + 计数器果冻弹跳，让"配置"这一步也有手感 |
| **答题页** | **⭐ 核心体验** —— 印章式对错反馈、试卷标签进度条、连胜火苗、纸屑庆祝动画 |
| 统计页 | 环形进度绘制动画 + 数字滚动递增，强化"成绩单"既视感 |

## 四、新增动效 / 规范细节

- **新增强调色**：高亮琥珀色 `#F59E0B`（用于连胜计数器、庆祝相关元素），其余色彩沿用原文档色彩系统。
- **动效时长规范**：微交互（按钮按下、选中）120–250ms；反馈类（盖章、翻转、纸屑）400–650ms；页面切换 300–320ms。避免任何装饰性动效超过 700ms，防止"等待感"。
- **无障碍与性能**：所有装饰性动效需遵循 `prefers-reduced-motion`，在用户开启该系统设置时缩短或跳过；对错反馈必须同时使用图标 + 文字 + 颜色三重信号，不能只靠颜色区分（色弱用户友好）。
- **盖章组件复用**：建议封装为独立组件（如 ArkUI 自定义组件 `StampFeedback`），接收 `correct: boolean` 参数，内部处理印章入场动画与墨点粒子，方便在答题页之外的场景（如打卡、签到）复用。
